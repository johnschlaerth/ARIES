You are building the ARIES MVP software.

Project:
ARIES = Automated Robotic Integrated Effects System.

Goal:
Build a local Python simulation that demonstrates AI-assisted orchestration of robotic ground effects nodes in a synthetic tactical environment. This is a concept demo only. It does not connect to real vehicles, real weapons, real radios, real jammers, or real sensors. All effects are abstract simulation state changes.

Primary demo requirements:
- Local Python app
- Pygame visualization
- Black-background 1980s tactical simulation style
- Synthetic contour map
- Friendly robotic ground nodes with different payload types
- Enemy air and ground entities
- Image classification module with mock mode and optional OpenAI API mode
- Threat level 1-10
- Central battle manager scoring
- Local vehicle-level scoring
- Terrain-aware movement for ground entities
- Straight-line drone movement
- Abstract effects only
- Dead/disabled entities remain visible as X
- Priority table
- Vectors
- Dotted paths
- Legend
- Event log
- Mission report
- Self-checking pytest testbenches

Critical instruction:
Build the simulator first in mock mode. Do not make OpenAI API calls required for the demo. API mode must be optional and must gracefully fail back to mock mode or cached results.

Visual style:
- black background
- contour lines
- red enemies
- blue/cyan/green/yellow friendlies
- simple geometric symbols
- visible vectors
- dotted planned paths
- side legend
- right-side priority table
- lower event log
- no fancy shading

Entity symbols:
Enemy drone = red downward triangle
Enemy ground = red diamond
Enemy EW = red circle with RF rings
Enemy unknown = red question-mark diamond
Friendly COMMS = blue hexagon / node-circle
Friendly EW = cyan circle with RF rings
Friendly ISR = green sensor/eye symbol
Friendly C-UAS = yellow turret symbol
Friendly EFFECT = green square/cross
Dead/disabled = X

Required controls:
SPACE = pause/resume
N = step one timestep
R = reset
F = fast mode
V = toggle vectors
P = toggle paths
S = save screenshot
Q or ESC = quit

Run modes:
mock = no API; deterministic; default
api = optional OpenAI vision classification; cached; graceful fallback
replay = replay saved event/state log

Classification rules:
The classifier must support enemy, friendly, neutral, non-threat, and unknown. It must not classify every image as a threat. A puppy, animal, civilian, random object, or friendly/allied military member should not become an enemy target.

Classification JSON schema:
{
  "name": "short object name",
  "allegiance": "friendly | enemy | neutral | unknown",
  "domain": "ground | air | fixed | non_threat",
  "entity_type": "enemy_drone | enemy_ground_vehicle | enemy_ew | enemy_infantry | enemy_unknown | friendly_human | friendly_vehicle | neutral_civilian | non_threat_animal | non_threat_object | unknown_contact",
  "threat_level": 1,
  "confidence": 0.0,
  "description": "one sentence",
  "rationale": "one sentence",
  "should_spawn_in_simulation": true
}

Scoring:
Global score should consider:
- threat level
- proximity to objective
- proximity to friendly units
- speed
- payload relevance
- confidence
- network threat

Local vehicle score should consider:
- enemy threat level
- proximity to this vehicle
- payload match
- path accessibility
- central manager recommendation
- risk penalty
- cooldown penalty

Effects:
All effects are abstract.
EW suppresses.
ISR classifies/reveals.
C-UAS neutralizes drones in abstract form.
EFFECT neutralizes ground/fixed threats in abstract form.
COMMS improves network support.
No real weapon, munition, or jamming logic.

Terrain:
Generate deterministic synthetic terrain using a seed.
Draw contour lines.
Ground entities use terrain-aware movement.
Air entities ignore terrain.

Reports:
At mission end, write:
- mission report TXT
- entity CSV
- event CSV
- summary JSON

Tests:
Create pytest tests for classification, scoring, pathfinding, effects, simulation stepping, report writing, and scenario loading. Tests must be deterministic and pass with no API key.

Acceptance:
The project is acceptable only if run_demo.py can launch a complete mock-mode simulation that runs reliably, displays moving entities, scores threats, assigns friendly units, applies abstract effects, shows dead entities as X, and generates reports.
